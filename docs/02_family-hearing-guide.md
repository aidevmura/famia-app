# ファミアアプリ 家族ヒアリングガイド
## 妻・子どもへのプロトタイプ体験フィードバック収集

---

> **このドキュメントについて**
> Notionに保存されている「Webサービス開発のヒアリング観点マスター（4層構造・12項目）」をベースに、
> ファミアアプリのプロトタイプ体験後の家族向けヒアリング用にカスタマイズしています。

---

## ヒアリングの目的

- 妻・子どもが実際に使ってみた感想を集める
- 使いにくい点・わかりにくい点を発見する
- 「あったらいいな」という追加要望を拾う
- 次の開発優先順位を決める材料にする

---

## 事前準備

1. スマホで `https://famia-app.vercel.app` を開いておく
2. デモアカウントでログインしておく（パパアカウント: papa@famia.app / password123）
3. きこちゃんのログインも試してもらう
4. 時間の目安: **妻：15〜20分、きこちゃん：5〜10分**

---

## Part 1｜きこちゃん（子ども）向けヒアリング

*※ 3〜6歳向けなので、言葉で聞かずに**行動観察**が中心です*

### 観察チェックリスト

| 観察ポイント | できた？ | メモ |
|-------------|---------|------|
| トップ画面で何をすればよいか直感的にわかったか | □ はい　□ いいえ | |
| ピクチャーパスワード（絵文字選択）でログインできたか | □ はい　□ いいえ | |
| 「今日のおしごと」のタスクを自分でタップできたか | □ はい　□ いいえ | |
| 「ひとりでできた！」ボタンの意味がわかったか | □ はい　□ いいえ | |
| お祝いアニメーションを見て喜んだか | □ はい　□ いいえ | |
| アバターショップで商品を見て反応したか | □ はい　□ いいえ | |
| アバターの着替えを自分でできたか | □ はい　□ いいえ | |

### 子どもに聞く質問（シンプルに）

- 「このゲーム、たのしかった？」（◯か✕で答えてもらう）
- 「どれがいちばん好きだった？」（画面を指差してもらう）
- 「難しかったところはあった？」
- 「お友達にも使ってほしい？」

## Part 2｜妻向けヒアリング

### ステップ1：第1層「目的・課題」を確認する

**背景と期待を理解する**

1. 今、子どものお手伝いや習慣管理で困っていることはある？
2. こういうアプリがあったら日常的に使いたいと思う？
3. このアプリを使うとしたら、どんな場面で使いたい？（朝の準備、夜の習慣など）
4. 子どもへの声がけが減るとしたら、どれくらい助かる？

---

### ステップ2：第2層「ユーザー視点」を確認する

**妻自身・子どもの使いやすさを探る**

5. 実際に触ってみて、直感的に使えたか？
6. わかりにくかった画面・ボタンはあった？（具体的にどこ？）
7. ログイン方法（パパ：メール/パスワード、子ども：絵文字）は使いやすかった？
8. スマホで見たときの見た目・文字の大きさは問題なかった？

---

### ステップ3：第3層「機能・コンテンツ」を確認する

**今ある機能への評価と追加要望**

9. 今の機能の中で「これはいい！」と思ったものは？
10. 今の機能の中で「これはいらない」と思ったものは？
11. 「これがないと使えない」と感じた機能はある？

**追加したい機能の候補（複数選択OK）**

- [ ] 子どもの成長記録・カレンダー表示
- [ ] 親からの「よくできたね！」メッセージ機能
- [ ] 週間・月間のポイントランキング
- [ ] タスク完了時に親にプッシュ通知
- [ ] 子どもが自分でタスクを提案できる機能
- [ ] 家族ルール・約束ごとの掲示板
- [ ] 複数の子ども（兄弟姉妹）への対応
- [ ] その他（自由記述）: 

---

### ステップ4：第4層「継続利用・運用」を確認する

**継続して使ってもらえるか**

12. このアプリ、1週間後も使っていると思う？理由は？
13. 毎日使うとしたら、どのタイミングで開く？（朝・夜・食事後など）
14. きこちゃんが「使いたい！」と言いそう？
15. アバターやポイントのシステムは、やる気につながりそう？
16. 有料（月額300〜500円程度）でも使いたいと思う機能はある？

## 参考：Notionヒアリング観点マスター（4層・12項目）

このガイドのベースになったNotionページ「04_Webサイトヒアリング観点マスター」の構造：

**4層のヒアリング構造**
- 第1層（目的）：なぜ作るのか / 何を達成したいのか
- 第2層（顧客）：誰に見てほしいのか / 何を伝えるべきか
- 第3層（実現手段）：どんなページ・機能・導線が必要か
- 第4層（運用）：誰が更新するのか / 公開後にどう成果を伸ばすのか

**12項目**
A. 相談背景　B. 事業理解　C. ターゲット　D. サイト目的　E. 現状把握　F. コンテンツ・構成
G. 機能要件　H. デザイン・ブランド　I. 運用・体制　J. 技術・法務　K. 予算・納期　L. 成功指標
